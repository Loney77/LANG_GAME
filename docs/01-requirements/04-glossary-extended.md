# Расширенный глоссарий (системный уровень)

Дополняет [бизнес-глоссарий Этапа 0](../00-project-charter/04-business-glossary.md)
системными и техническими терминами (всего ≥ 20 новых).

| # | Термин | Определение |
|---|--------|-------------|
| 1 | **JWT** | JSON Web Token - подписанный токен для аутентификации без сессий на сервере. |
| 2 | **Роль (Role)** | Признак прав доступа пользователя: `USER` или `ADMIN`. |
| 3 | **BCrypt** | Алгоритм хэширования паролей с солью; хранится только хэш. |
| 4 | **REST API** | Стиль взаимодействия клиент-сервер по HTTP с ресурсами и стандартными методами. |
| 5 | **Эндпоинт** | Конкретный URL+метод REST API (напр. `POST /api/auth/login`). |
| 6 | **OpenAPI / Swagger UI** | Спецификация и веб-интерфейс документации REST API. |
| 7 | **DTO** | Data Transfer Object - объект передачи данных между клиентом и сервером. |
| 8 | **Entity (JPA)** | Класс, отображаемый на таблицу БД. |
| 9 | **Репозиторий** | Компонент Foundation для доступа к данным сущности. |
| 10 | **PCMEF** | Слоевой архитектурный паттерн: Presentation-Control-Mediator-Entity-Foundation. |
| 11 | **Токенизатор алфавита** | Алгоритм разбиения слова на буквы карач. алфавита с учётом диграфов. |
| 12 | **Целевое слово** | Загаданное слово в задании Сёздл. |
| 13 | **Статус плитки** | Результат сравнения буквы догадки: на месте / есть / отсутствует. |
| 14 | **Payload задания** | JSON-данные конкретного задания (набор букв, сетка кроссворда). |
| 15 | **Сессия игры (GameSession)** | Запись одного прохождения задания с результатом и очками. |
| 16 | **Анти-чит** | Принцип: проверка ответов и подсчёт очков только на сервере. |
| 17 | **Оффлайн-кэш** | Локальное хранилище (AsyncStorage) для работы без сети. |
| 18 | **Flyway** | Инструмент версионирования и миграций схемы БД. |
| 19 | **3НФ** | Третья нормальная форма - критерий нормализации схемы БД. |
| 20 | **JaCoCo** | Инструмент измерения покрытия Java-кода тестами. |
| 21 | **Data Mapper** | Паттерн: отделение доменных объектов от логики доступа к данным. |
| 22 | **Identity Map** | Паттерн: единый экземпляр объекта на сессию (контекст персистентности). |
