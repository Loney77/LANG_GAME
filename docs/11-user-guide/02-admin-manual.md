# Руководство администратора

## 1. Назначение

Администратор управляет содержимым приложения: словарём и тематическими
категориями слов. Управление выполняется через защищённые REST-эндпоинты сервера;
операции изменения доступны только пользователям с ролью `ADMIN`.

## 2. Роли и доступ

| Роль | Права |
|------|-------|
| `USER` | Игра, просмотр словаря и рейтинга, ведение собственных результатов |
| `ADMIN` | Всё перечисленное и операции создания, изменения и удаления слов и тем |

Аутентификация выполняется по JWT. Токен выдаётся при входе и передаётся в
заголовке `Authorization: Bearer <token>`. Запрос без токена к защищённому ресурсу
возвращает `401`, запрос с ролью `USER` к административной операции — `403`.

## 3. Получение токена администратора

```http
POST /api/auth/login
Content-Type: application/json

{ "username": "admin", "password": "<пароль>" }
```

Ответ содержит JWT, который используется в последующих запросах.

## 4. Управление словарём

| Операция | Метод и путь | Доступ |
|----------|--------------|--------|
| Поиск и фильтрация слов | `GET /api/words?theme=&length=` | USER |
| Получение слова по идентификатору | `GET /api/words/{id}` | USER |
| Добавление слова | `POST /api/words` | ADMIN |
| Изменение слова | `PUT /api/words/{id}` | ADMIN |
| Удаление слова | `DELETE /api/words/{id}` | ADMIN |

Пример добавления слова:

```http
POST /api/words
Authorization: Bearer <token>
Content-Type: application/json

{ "text": "къол", "translation": "рука", "themeId": 3 }
```

Длина слова в карачаевских буквах вычисляется сервером автоматически с учётом
диграфов; задавать её вручную не требуется.

## 5. Управление темами

| Операция | Метод и путь | Доступ |
|----------|--------------|--------|
| Список тем | `GET /api/themes` | USER |

Темы используются для группировки слов и фильтрации при подборе заданий.

## 6. Документация API

Полное описание эндпоинтов, схем запросов и ответов доступно в Swagger UI:
`http://<адрес-сервера>:8137/swagger-ui.html`. Спецификация OpenAPI —
`http://<адрес-сервера>:8137/v3/api-docs`.

## 7. Эксплуатация сервера

Параметры сервера задаются переменными окружения:

| Переменная | Назначение | Значение по умолчанию |
|------------|-----------|------------------------|
| `SERVER_PORT` | Порт HTTP | `8137` |
| `DB_URL` | JDBC-адрес базы данных | `jdbc:postgresql://localhost:5433/langgame` |
| `DB_USER` | Пользователь БД | `langgame` |
| `DB_PASSWORD` | Пароль БД | `langgame` |
| `JWT_SECRET` | Ключ подписи токенов | значение для разработки |

В промышленной среде значения `JWT_SECRET` и учётных данных БД задаются
индивидуально. Схема базы данных и начальный словарь применяются автоматически
миграциями Flyway при первом запуске.
