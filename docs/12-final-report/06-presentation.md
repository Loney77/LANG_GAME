# «Сёз оюн»

### Мобильное приложение для обучения карачаевскому языку в игровой форме

Курсовой проект по дисциплине «Программная инженерия»
Траектория В: React Native + Java Spring Boot

Урусов Руслан, ПИЖ-б-о-23-2

---

## Проблема

- Карачаевский язык — с сокращающейся сферой использования.
- Современных интерактивных инструментов для изучения практически нет.
- Существующие словари не интерактивны и не мотивируют к регулярной практике.

---

## Решение

Мобильное приложение со словесными играми на карачаевской лексике:

- **Сёздл**, **Анаграммы**, **Викторина**, **Кроссворд**
- Ежедневные задания, очки и рейтинг игроков
- Серверная проверка ответов и управление словарём администратором

---

## Цели проекта

1. Интерактивный инструмент расширения словарного запаса.
2. Вовлечённость через задания, очки и лидерборд.
3. Удобное управление словарём для администратора.

---

## Архитектура PCMEF

Зависимости строго сверху вниз: Presentation → Control → Mediator → Entity → Foundation

| Слой | Реализация |
|------|-----------|
| Presentation | React Native |
| Control | Контроллеры Spring |
| Mediator | Бизнес-логика игр |
| Entity | JPA-сущности + токенизатор |
| Foundation | Репозитории + Data Mapper |

---

## Технологический стек

- **Клиент:** TypeScript, React Native, React Navigation, Axios, AsyncStorage
- **Сервер:** Java 17, Spring Boot 3, Spring Data JPA, Spring Security + JWT, springdoc-openapi
- **База данных:** PostgreSQL (3НФ), Flyway
- **Тестирование:** JUnit 5, Mockito, JaCoCo

---

## Модель данных

- Сущности: `User`, `Theme`, `Word`, `GameType`, `Puzzle`, `QuizQuestion`, `GameSession`
- Третья нормальная форма, ограничения целостности
- Схема и начальные данные — миграции Flyway

---

## Ключевое решение: токенизатор алфавита

- Диграфы (гъ, къ, нъ, нг, дж) считаются одной буквой.
- Используется в подсчёте длины слова, подсветке «Сёздл», анаграммах.
- Покрыт модульными тестами в первую очередь.

---

## Ключевое решение: серверная проверка

- Правильные ответы хранятся только на сервере.
- Клиент получает лишь результат проверки.
- Исключены раскрытие ответа и обход правил игры.

---

## Паттерны проектирования

- **Data Mapper** — отделение домена от DTO и персистентности.
- **Identity Map** — контекст персистентности Hibernate.
- **Lazy Load** — отложенная загрузка связей.

---

## Программный интерфейс

- Более восьми REST-эндпоинтов.
- Документация OpenAPI и Swagger UI.
- Аутентификация по JWT, разграничение ролей (USER / ADMIN).

---

## Тестирование

- Модульные тесты ключевой логики: токенизатор, проверка «Сёздл», подсчёт очков.
- Покрытие серверного кода — свыше 40% (JaCoCo).
