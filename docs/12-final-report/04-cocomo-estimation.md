# Оценка трудоёмкости по модели COCOMO

Оценка выполнена по базовой модели COCOMO (Constructive Cost Model) на основе
фактического объёма исходного кода проекта.

## 1. Объём исходного кода

| Компонент | Строк кода |
|-----------|-----------|
| Сервер (Java, основной код) | 2837 |
| Клиент (TypeScript / React Native) | 1133 |
| Схема и миграции базы данных (SQL) | 373 |
| **Итого (учитывается в оценке)** | **≈ 4343** |
| Модульные тесты (Java) | 413 |

Размер для оценки: **KLOC ≈ 4.3** (тысяч строк кода; тесты в базовую оценку
трудоёмкости не включаются).

## 2. Выбор класса проекта

Проект относится к типу **Organic** (распространённый): небольшая команда,
хорошо понятные требования, широкое использование зрелых фреймворков
(Spring Boot, React Native). Коэффициенты базовой модели для этого класса:

| Параметр | a | b | c | d |
|----------|---|---|---|---|
| Organic | 2.4 | 1.05 | 2.5 | 0.38 |

## 3. Расчёт

**Трудоёмкость (человеко-месяцы):**

```
PM = a · (KLOC)^b = 2.4 · (4.3)^1.05 ≈ 2.4 · 4.63 ≈ 11.1 чел.-мес.
```

**Длительность разработки (месяцы):**

```
TDEV = c · (PM)^d = 2.5 · (11.1)^0.38 ≈ 2.5 · 2.50 ≈ 6.2 мес.
```

**Средняя численность исполнителей:**

```
N = PM / TDEV = 11.1 / 6.2 ≈ 1.8 чел.
```

## 4. Результаты и сопоставление с фактом

| Показатель | Оценка COCOMO | Факт по проекту |
|-----------|---------------|------------------|
| Трудоёмкость | ≈ 11.1 чел.-мес. | 1 исполнитель |
| Длительность | ≈ 6.2 мес. | 4.5 мес. (18 недель) |
| Численность | ≈ 1.8 чел. | 1 чел. |

Фактическая длительность меньше оценки COCOMO. Основные причины расхождения:

- высокий уровень повторного использования готовых компонентов фреймворков
  (аутентификация, ORM, навигация, генерация документации API), что сокращает
  объём оригинального кода и трудоёмкость;
- стабильные и заранее зафиксированные требования, не потребовавшие переработок;
- модель COCOMO ориентирована на промышленную разработку с полной занятостью и
  даёт консервативную (завышенную для учебного проекта) оценку.

Оценка подтверждает реалистичность выбранного объёма работ для выполнения силами
одного исполнителя в отведённый срок.
