# ADR-001. Применение архитектурного паттерна PCMEF

- **Статус:** Принято
- **Дата:** 2026-06

## Контекст

Курсовой проект требует слоевой архитектуры с чётким разделением
ответственности и контролируемыми зависимостями. Альтернативы: «луковая»/
Clean Architecture, классический MVC, гексагональная.

## Решение

Используем **PCMEF** (Presentation-Control-Mediator-Entity-Foundation) -
обязательный паттерн методички. Зависимости направлены строго сверху вниз,
слои общаются через интерфейсы `IService` и `IRepository`.

## Последствия

- **+** Каждый слой тестируется изолированно (моки интерфейсов).
- **+** Соответствие требованиям курса; снимается риск штрафа −20%.
- **+** Изоляция бизнес-логики (Mediator) от транспорта (Control).
- **−** Больше «церемоний»: интерфейсы и DTO добавляют классов.
- Маппинг на пакеты: `control`, `mediator`, `entity`, `foundation`, `config`.
