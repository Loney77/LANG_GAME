# ADR-004. Stateless-аутентификация через JWT

- **Статус:** Принято
- **Дата:** 2026-06

## Контекст

Мобильному клиенту нужна аутентификация. Варианты: серверные сессии (cookie) или
stateless-токены (JWT). Мобильные клиенты плохо дружат с cookie-сессиями и
горизонтальным масштабированием.

## Решение

Используем **JWT**: при `login`/`register` сервер выдаёт подписанный токен с
именем пользователя и ролью; клиент хранит его и шлёт в заголовке `Authorization:
Bearer`. Проверка - фильтром Spring Security. Пароли хранятся как **BCrypt**-хэши.
Роли: `USER`, `ADMIN`.

## Последствия

- **+** Сервер не хранит состояние сессий; просто масштабируется.
- **+** Единый механизм для мобильного клиента и (опц.) веб-админки.
- **−** Сложнее немедленный отзыв токена (смягчается коротким сроком жизни; refresh-токены - вне MVP).
- Секрет подписи - через переменную окружения `JWT_SECRET` (не в репозитории).
