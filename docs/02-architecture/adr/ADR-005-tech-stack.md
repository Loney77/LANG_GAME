# ADR-005. Технологический стек и сборка

- **Статус:** Принято
- **Дата:** 2026-06

## Контекст

Траектория В фиксирует серверную часть на Java + Spring Boot и клиент на одном из
кроссплатформенных стеков. Нужно выбрать конкретные версии и инструмент сборки.

## Решение

- **Клиент:** React Native + TypeScript (экосистема JS, кроссплатформенность).
- **Сервер:** Java 17 (toolchain), **Spring Boot 3.5.14**, **Gradle 9.3.1**
  (актуальные стабильные версии; Gradle 9 совместим с Boot 3.5).
- **БД:** PostgreSQL + Flyway (версионирование схемы).
- **Документация API:** springdoc-openapi (Swagger UI).
- **Тесты/покрытие:** JUnit 5 + JaCoCo (порог 40% в `build.gradle`).

Сборка вынесена в `server/` (monorepo: `server/`, `mobile/`, `frontend/`,
`docker/`, `docs/`). Версия Java - 17 через Gradle toolchain (хотя в системе JDK 21),
для соответствия базовой версии методички и стабильности.

## Последствия

- **+** Актуальный стек, единый репозиторий, воспроизводимая сборка (wrapper).
- **+** Порог покрытия автоматически проверяется (`jacocoTestCoverageVerification`).
- **−** Gradle 9 даёт предупреждения о будущей несовместимости с Gradle 10 (от плагина
  dependency-management) - на сборку не влияет, отслеживаем при апгрейдах.
