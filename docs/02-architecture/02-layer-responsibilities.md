# Ответственность слоёв PCMEF

| Слой | Отвечает за | НЕ должен | Реализация |
|------|-------------|-----------|------------|
| **Presentation** | UI, ввод, отображение, состояния загрузки/ошибки, оффлайн-кэш | Содержать бизнес-логику игр | React Native (TS) |
| **Control** | Приём HTTP-запросов, валидация DTO, маппинг, коды ответов, проверка ролей | Обращаться к БД напрямую; содержать бизнес-правила | `@RestController` |
| **Mediator** | Бизнес-логика игр, транзакции, координация, подсчёт очков | Знать о деталях HTTP/Presentation | `@Service` + `@Transactional` |
| **Entity** | Состояние и доменные методы (`getLetters`, `calculateScore`, `isExpired`) | Содержать логику доступа к данным | JPA `@Entity` |
| **Foundation** | Доступ к данным, запросы, мапперы, токенизатор | Содержать бизнес-правила | Spring Data JPA + Data Mapper |

## Правило зависимостей

```
Presentation → Control → Mediator → Entity → Foundation
```

- Зависимости только сверху вниз; обратных и циклических связей нет.
- Control зависит от Mediator **через интерфейс** `IService`, а не от конкретного класса.
- Mediator зависит от Foundation **через интерфейс** `IRepository`.
- Entity не зависит ни от кого выше; используется всеми нижними слоями как данные.

## Проверка соблюдения (для код-ревью Этапа 6)

1. В `control` нет импортов из `jakarta.persistence` / репозиториев напрямую.
2. В `entity` нет импортов из `control`/`mediator`.
3. Бизнес-вычисления (очки, статусы плиток) находятся в `mediator`/`entity`, не в `control`.
4. Граф зависимостей пакетов ацикличен (см. [04-dependency-diagram.md](04-dependency-diagram.md)).
