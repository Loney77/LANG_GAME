## 2. Бизнес-контекст

**Проблема.** Карачаевский язык относится к языкам с сокращающейся сферой
использования: молодёжь всё реже владеет им активно, а доступных современных
цифровых инструментов для его изучения почти нет. Существующие словари -
бумажные либо неструктурированные, не интерактивные и не мотивируют к
регулярной практике.

**Решение.** Мобильное приложение со словесными играми (Сёздл, анаграммы,
викторина, кроссворд) на карачаевской лексике. Игровая форма и ежедневные
задания формируют привычку к ежедневной практике, а серверная проверка ответов
и лидерборд добавляют соревновательную мотивацию. Контентом (словарём) управляет
администратор через защищённый интерфейс.

## 3. Цели проекта

1. Предоставить пользователю интерактивный инструмент для расширения словарного
   запаса карачаевского языка через игры.
2. Обеспечить вовлечённость через ежедневные задания, систему очков и лидерборд.
3. Дать администратору удобное управление словарём и игровым контентом (CRUD).

## 4. Ключевые показатели успеха

| Показатель | Целевое значение |
|------------|------------------|
| Объём игрового словаря | ≥ 240 слов в базе на момент сдачи (рост через админ-CRUD) |
| Доля слов, пригодных для Сёздл (5 букв) | ≥ 200 слов |
| Покрытие серверного кода тестами | > 40% (JaCoCo) |
| Число REST-эндпоинтов с документацией OpenAPI | ≥ 8 |
| Время отклика проверки догадки Сёздл | < 300 мс |

## 5. Границы проекта (Scope)

**В составе MVP:** регистрация/вход (JWT), игры Сёздл, анаграммы, викторина;
ежедневное задание; подсчёт очков и лидерборд; админ-CRUD словаря и тем;
оффлайн-кэш словаря и заданий.

**Stretch / вне MVP:** кроссворд (реализуется последним); обучающий слой
(достижения, интервальное повторение); веб-админка; публикация в Google Play.

## 6. Ключевые риски

| Риск | Стратегия реагирования |
|------|------------------------|
| Малый и однобуквенный (а/б) объём словаря | Использовать `vocab.txt` для MVP; нарастить через админ-CRUD; запланировать импорт из словаря Гочияевой/Суюнчева |
| Диграфы (гъ, къ, нъ, нг, дж) ломают логику Сёздл и подсчёт длины | Серверный токенизатор алфавита, покрытый юнит-тестами в первую очередь |
| Кроссворд трудоёмок и рискует по срокам | Делать последним; при дефиците времени исключить (3 игр достаточно для 5+ экранов) |
| Протекание бизнес-логики в контроллеры (нарушение PCMEF, −20%) | Держать логику строго в `@Service`; код-ревью графа зависимостей |
