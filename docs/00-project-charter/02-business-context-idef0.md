# Диаграмма бизнес-контекста (IDEF0, уровень A-0)

Контекстная диаграмма верхнего уровня описывает основную бизнес-функцию системы
как «чёрный ящик» с входами, выходами, управлением и механизмами (нотация ICOM).

## A-0: Обучать карачаевскому языку в игровой форме

```
                    Управление (Controls)
          ┌───────────────────┬────────────────────┐
          │ Правила игр       │ Нормы карачаевского │
          │ (Сёздл, квиз…)   │ языка / орфография  │
          ▼                   ▼
   ┌─────────────────────────────────────────────────┐
   │                                                   │
 Входы (Inputs)                                  Выходы (Outputs)
──────────────►│   A0: ОБУЧАТЬ КАРАЧАЕВСКОМУ      │──────────────►
 • Действия     │       ЯЗЫКУ В ИГРОВОЙ ФОРМЕ      │  • Освоенная
   игрока       │                                   │    лексика
 • Догадки/     │                                   │  • Очки, место
   ответы       │                                   │    в лидерборде
 • Словарные    │                                   │  • Статистика
   статьи (от   │                                   │    игр
   админа)      │                                   │  • Обновлённый
                │                                   │    словарь
   ┌─────────────────────────────────────────────────┐
          ▲                   ▲
          │ Мобильное         │ Сервер (Spring Boot),│
          │ приложение (RN)   │ PostgreSQL, JWT      │
          └───────────────────┴────────────────────┘
                    Механизмы (Mechanisms)
```

## Описание элементов ICOM

| Категория | Элементы |
|-----------|----------|
| **Входы (I)** | Действия игрока, догадки в Сёздл, ответы викторины, словарные статьи от администратора |
| **Управление (C)** | Правила игр; нормы и орфография карачаевского языка (в т.ч. алфавит с диграфами); политика ролей и доступа |
| **Выходы (O)** | Освоенная лексика; начисленные очки и позиция в лидерборде; статистика сыгранных партий; пополненный/исправленный словарь |
| **Механизмы (M)** | Мобильное приложение (React Native); сервер на Spring Boot; БД PostgreSQL; механизм аутентификации JWT |
