# Применённые паттерны проектирования

Перечень паттернов GoF и корпоративных паттернов (PoEAA), реализованных в коде,
с указанием места и назначения. Три паттерна рефакторинга (Data Mapper, Identity
Map, Lazy Load) подробно разобраны на [этапе 6](../07-refactoring/README.md).

## Корпоративные паттерны (PoEAA)

| Паттерн | Где в коде | Назначение |
|---------|-----------|------------|
| **Repository** | `foundation/I*Repository` (Spring Data JPA) | Абстракция доступа к данным; Mediator работает с коллекцией доменных объектов, не зная о SQL |
| **Data Mapper** | `mediator/WordMapper`, записи `mediator/dto/*` | Разделение доменных сущностей и DTO; сущности не «протекают» в REST-контракт. См. [01-data-mapper.md](../07-refactoring/01-data-mapper.md) |
| **Identity Map** | Контекст персистентности Hibernate (L1-кэш) | В рамках транзакции одна строка БД = один объект в памяти. См. [02-identity-map.md](../07-refactoring/02-identity-map.md) |
| **Lazy Load** | `@ManyToOne(fetch = LAZY)` на связях сущностей | Связанные данные загружаются по требованию. См. [03-lazy-load.md](../07-refactoring/03-lazy-load.md) |
| **DTO** | `mediator/dto/*` (record-классы) | Передача данных между клиентом и сервером отдельно от доменной модели |
| **Service Layer** | `mediator/*Service` за интерфейсами `I*Service` | Слой бизнес-операций с транзакциями, единая точка входа для Control |

## Паттерны GoF

| Паттерн | Где в коде | Назначение |
|---------|-----------|------------|
| **Strategy** | `ScoringService` (алгоритм очков); полиморфные `I*Service` игр за единым `GameController` | Взаимозаменяемые алгоритмы: правила подсчёта очков и логика проверки для каждой игры инкапсулированы отдельно |
| **Facade** | `*Service` поверх репозиториев, доменных утилит и `GameSessionRecorder` | Контроллер вызывает один метод сервиса, скрывающий координацию нескольких объектов |
| **Chain of Responsibility** | Цепочка фильтров Spring Security, звено `JwtAuthFilter` | Запрос проходит через цепочку обработчиков (аутентификация по JWT до контроллера) |
| **Singleton** | Бины Spring (сервисы, репозитории, конфигурация) | Единственный управляемый контейнером экземпляр на приложение |
| **Dependency Injection (IoC)** | Конструкторное внедрение (`@RequiredArgsConstructor`) во всех слоях | Слои зависят от интерфейсов; реализации подставляет контейнер — слабая связанность |

## Доменные алгоритмы

Не паттерны GoF, но ключевые инкапсулированные алгоритмы ядра:

- **`AlphabetTokenizer`** — разбиение слова на буквы карачаевского алфавита, где
  диграфы (гъ, къ, нъ, нг, дж) считаются одной буквой.
- **`LetterMatcher`** — двухпроходная подсветка букв догадки (CORRECT/PRESENT/ABSENT)
  с корректной обработкой повторяющихся букв.
- **`CrosswordGenerator`** — жадное размещение слов с пересечениями по общим буквам
  и проверкой конфликтов клеток (чистый алгоритм, покрыт юнит-тестом).
- **`AnagramShuffler`** — перемешивание букв слова с учётом диграфов.
